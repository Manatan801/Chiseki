# 地籍調査DXツール - プロジェクト仕様書

## 概要
JWCADからDXFエクスポートされた地籍調査図面を解析し、以下2つの帳票を自動生成するPythonツール。

1. **結線指示票** - 各筆（土地）の境界杭番号を左上から時計回りに一周させたリスト
2. **交点計算指示書** - 公共用地境界線と民有地境界線の交点を特定する指示書

## DXFファイル構造（解析済み）

### レイヤー構成
| Layer | 用途 | 主なエンティティ |
|-------|------|-----------------|
| **34** | 上部エリアの杭・境界 | POINT×129, CIRCLE×138(r=0.25), MTEXT×230, LWPOLYLINE×357, ARC×366 |
| **49** | 下部エリアの杭・境界 | POINT×68, CIRCLE×74(r=0.25), MTEXT×140, LWPOLYLINE×249, ARC×159 |
| **35** | 既存杭マーカー？ | CIRCLE×27(r=0.5/0.625), LWPOLYLINE×26 |
| **36** | 補助点？ | ARC×26, CIRCLE×13(r=0.25), POINT×13 |
| **37** | 補助マーカー？ | CIRCLE×12(r=0.5) |
| **50** | 補助点？ | CIRCLE×15(r=0.5), POINT×15 |
| **51** | 補助マーカー？ | CIRCLE×10(r=0.625), LWPOLYLINE×10 |
| **52** | 補助要素 | ARC×30, CIRCLE×15(r=0.25), LWPOLYLINE×2 |
| **53** | 所有者名テキスト | MTEXT×29 |
| **2305** | 面積ラベル + 外周線 | LWPOLYLINE×3100, CIRCLE×32, TEXT×5 |

### エンティティ構造の詳細

#### 杭（境界点）
- **主要杭**: POINT（Layer 34/49）+ CIRCLE（r=0.25, 同Layer）
- **追加杭（5000番台等）**: CIRCLE（r=0.25）のみで、POINTエンティティがない場合がある
  - Layer 36: CIRCLE×13 (r=0.25) + POINT×13 → 1.5048, 1.5049等
  - Layer 50: CIRCLE×15 (r=0.5) + POINT×15 → 1.5042, 1.5038等のマーカー（r=0.5は別種）
  - Layer 52: CIRCLE×15 (r=0.25) + POINTなし → 追加杭の座標はCIRCLE中心から取得
- **★重要**: 杭座標の収集は `r=0.25のCIRCLE中心`を全レイヤーから取るのが最も確実
  - POINTだけに頼るとLayer 52の杭が抜け落ちる
  - CIRCLE(r=0.25)をキーにすれば全杭を網羅できる

#### 杭番号テキスト
- **MTEXT**エンティティ（Layer 34/49）
- フォーマット: `\W0.50;{杭番号}` （例: `\W0.50;1.777`, `\W0.50;-1.832`）
- テキスト位置はPOINTから距離0.25〜3.3の範囲に配置されている
- 抽出正規表現: `re.sub(r'\\[WwHhAaCcFfPpQqTtLlOo][^;]*;', '', raw_text).strip()`

#### 杭番号の命名規則
- 通常の杭: `{系統番号}.{連番}` (例: 1.777, 10.486, 13.289)
- 交点（計算点）: `-{系統番号}.{連番}` (例: -1.832, -10.633, -13.298)
- 5000番台の杭: `{系統番号}.5{連番}` (例: 1.5017, 13.5011) ← 追加杭と推測

#### 地番テキスト
- MTEXT（Layer 34/49）で杭番号と同一レイヤーに混在
- 識別方法: `re.match(r'^\d+(-\d+)?$', text)` かつ小数点を含まない
- 例: 1070-6, 1069-5, 490-1, 143

#### 地目テキスト
- MTEXT（Layer 34/49）、フォント幅指定 `\W0.83;`
- 例: 宅地, 山林, 雑種地, 公衆用道路

#### 公共用地ラベル
- MTEXT（Layer 34/49）、フォント幅指定 `\W0.83;`
- パターン: `県道-{番号}`, `河川-{番号}`, `市道-{番号}`, `水-{番号}`, `道-{番号}`
- 例: 県道-5, 河川-2, 市道-19, 水-23

#### 境界線
- **LWPOLYLINE**（2点の線分、閉じていない）が境界線
- 端点がPOINT座標と一致するものが筆界線
- Layer 34: 357本, Layer 49: 249本
- **重要**: 閉合ポリラインではなく、個別の2点線分の集合

### テキスト種別の識別方法
```python
import re

def classify_text(raw_text):
    text = re.sub(r'\\[WwHhAaCcFfPpQqTtLlOo][^;]*;', '', raw_text).strip()
    
    # 交点杭番号（マイナス接頭辞）
    if re.match(r'^-\d+\.\d+$', text):
        return 'intersection_stake', text
    
    # 通常の杭番号
    if re.match(r'^-?\d+\.\d+$', text):
        return 'stake', text
    
    # 筆界点コード（P3P3-Fxxxx-x形式）
    if text.startswith('P3P3-'):
        return 'survey_code', text
    
    # 地番（整数 or 整数-整数）
    if re.match(r'^\d+(-\d+)?$', text):
        return 'parcel_number', text
    
    # 公共用地ラベル
    if re.match(r'^(県道|河川|市道|国道|水|道)-', text):
        return 'public_land', text
    
    # 地目
    if text in ['宅地', '山林', '雑種地', '公衆用道路', '田', '畑']:
        return 'land_use', text
    
    # 所有者名（Layer 53で別途取得）
    return 'other', text
```

## 結線指示票の仕様

### 出力フォーマット
Excelファイル（.xls）。1筆につき1シート。

### シート構造
```
Row 0: ヘッダー「一 筆 地 境 界 点 結 線 指 示 票」
Row 1: 年度、地区名、記入者
Row 2: カラムヘッダー
Row 3: ブロック番号、大字名、字名、地番
Row 4: 所有者名、調査日、決定日
Row 5〜: 番号列(Col 0) + 杭番号列(Col 1) ← ここが自動生成対象
```

### 結線ルール（正解データから確認済み）
- 各筆の境界杭番号を**左上から時計回り**に一周
- 最後は最初の杭番号に戻る（閉合）
- 交点（-付き）も含む

## 交点計算指示書の仕様

### 概念
- 道路・河川等の公共用地境界杭同士を結んだ線 = **基準線**（確定済み・不変）
- 民有地の境界から延長した線が基準線と交差する点 = **交点**（杭番号に-接頭辞）
- 指示書は、交点ごとに「基準線の2点」「延長線の2点」を記録

### Excelフォーマット（解析済み）
- 3列×3行のブロックが繰り返し配置（横に3ブロック）
- 各ブロック構造:
  - Row +0: ○ = 基準線の点1
  - Row +3: ○ = 基準線の点2  
  - Row +5: ○(左) = 延長線の点1, ◎(中央) = 交点, ○(右) = 延長線の点2
- 記入済みサンプル（Row 5, Row 12, Block 13ブロック）:
  - 基準線点: 13.193 / 13.239 / 13.297
  - 交点: -13.299 / -13.238 / -13.298
  - 延長線関連点: 13.194, 13.5009, 13.185, 13.187, 13.189 等

## アルゴリズム設計

### Phase 1: DXF解析（共通基盤）

```
1. DXFファイル読み込み（ezdxf）
2. 全CIRCLE(r=0.25)の中心座標を抽出（全レイヤー）→ 杭座標リスト
   ★ POINTだけでなくCIRCLE(r=0.25)を使うことで5000番台の追加杭も網羅
3. 全MTEXTを抽出し分類（杭番号/地番/地目/公共用地ラベル/所有者名）
4. CIRCLE中心 → 最近傍杭番号テキストをマッチング
   - 距離閾値: 5.0以内（サンプルでは0.25〜3.5）
   - 同一レイヤー優先でマッチングし、見つからなければ近接レイヤーも検索
5. 全LWPOLYLINE（2点線分）を抽出
6. 線分の端点 → 杭座標をマッチング（距離閾値: 0.1）
7. 接続グラフを構築（杭番号をノード、線分をエッジ）
```

### Phase 2: 結線指示票生成

```
1. 地番テキストの位置を基に、各地番に属する杭を特定
   - 地番テキストの座標を含む最小の閉合ポリゴンを構築
   - または、地番テキストの最近傍の杭グループを特定
2. 各筆について:
   a. 筆に属する杭のサブグラフを抽出
   b. 閉合パス（オイラー閉路）を探索
   c. 左上の杭を起点に設定
   d. 時計回り方向でパスを整列
3. Excelテンプレートに書き込み
```

### Phase 3: 交点計算指示書生成

```
1. 交点杭（-接頭辞）をリストアップ
2. 各交点について:
   a. 交点杭の座標を取得
   b. 交点を通る2本の線分を特定:
      - 基準線（公共用地境界の杭同士を結ぶ線）
      - 延長線（民有地の杭同士を結ぶ線の延長）
   c. 基準線の2点、延長線の2点を記録
3. Excelテンプレートに書き込み
```

## 開発計画

### Step 1: 基盤モジュール (`dxf_parser.py`)
- DXF読み込み、エンティティ抽出、テキスト分類
- 杭座標↔杭番号マッチング
- 境界線グラフ構築

### Step 2: 結線生成 (`kessen_generator.py`)
- 筆ごとの杭グループ特定
- 閉合パス探索
- 時計回り整列
- 正解データとの照合テスト

### Step 3: 交点生成 (`kouten_generator.py`)
- 交点の基準線/延長線特定
- 指示書フォーマット出力

### Step 4: Excel出力 (`excel_writer.py`)
- テンプレートへの書き込み
- 結線指示票/交点計算指示書の出力

## 注意事項
- JWCADからDXF変換時に文字化けやレイヤー情報欠落の可能性あり。DXF保存は「通常のDXF保存」で（SXF対応拡張ではない）
- 杭番号テキストとPOINTの距離は一定ではない（0.25〜3.3）。最近傍マッチングで対応
- Layer 34とLayer 49はエリアが異なるが同じ処理ロジックで対応可能
- 閉合ポリラインがないため、線分端点の接続グラフから閉路を探索する必要がある
