# Chiseki - 地籍調査DX帳票自動生成業務効率化ツール

JW-CADで作成した地籍調査のDXF図面から、**結線指示票**と**交点計算指示書**を自動生成するPythonツールです。
※注意点　生成されたデータを必ず人が確認することを前提として作成したツールです。
　　　　　そのため、自動生成されたデータは必ず人の手で確認・修正し、正確なものに仕上げてください。

## これで何が変わるか

| | Before（手作業） | After（Chiseki） |
|---|---|---|
| 結線指示票 | 図面を見ながら1筆ずつ杭番号を書き写す | DXFを読み込むだけで全筆一括生成 |
| 交点計算指示書 | 交点ごとに基準線・方向点を図面から読み取る | 同一直線ペア検出で自動判定 |
| 所要時間 | 1班あたり数時間 | 数秒 |

## 動作環境

- Python 3.10以上
- Windows / macOS / Linux（WSL含む）

## セットアップ

```bash
git clone https://github.com/yourname/Chiseki.git
cd Chiseki
python -m venv venv
source venv/bin/activate    # Windows: venv\Scripts\activate
pip install -r requirements.txt
```

## 使い方

### 基本（全地番を処理）

```bash
python -m src.main "data/input/対象ファイル.DXF"
```

### オプション

```bash
# 特定の地番のみ処理
python -m src.main "対象.DXF" --parcels 1070-6,1069-5,1069-7

# 結線指示票のみ生成（交点計算なし）
python -m src.main "対象.DXF" --no-kouten

# 交点計算指示書のみ生成（結線なし）
python -m src.main "対象.DXF" --no-kessen

# 出力先を変更
python -m src.main "対象.DXF" --output-dir results

# テンプレートを指定
python -m src.main "対象.DXF" --kessen-template "自前テンプレート.xls"
```

### 出力ファイル

| ファイル | 内容 |
|---|---|
| `output/結線指示票.xls` | 各筆の杭番号リスト（北西角から時計回り） |
| `output/交点計算指示書.xlsx` | 交点杭の基準線・方向点 |

## DXF図面の作成ルール

本ツールは以下のルールで作成されたDXFファイルを前提としています。

### 必須要素

| 要素 | DXFエンティティ | 条件 |
|---|---|---|
| 杭（境界点） | CIRCLE | **半径 = 0.25** |
| 杭番号 | MTEXT | 通常杭: `1.830` / 交点杭: `-1.824` |
| 境界線 | LWPOLYLINE | **頂点数 = 2**（2点の線分） |
| 地番 | MTEXT | `1070-6` 形式（整数 or 整数-整数） |
| 公共用地名 | MTEXT | `県道-5`, `市道-19`, `水-23` 等 |

### テキスト配置の注意点

- 杭番号テキストは対応するCIRCLE（r=0.25）の近くに配置してください
- 地番テキストは該当区画の内部に配置してください
- 狭い区画で引き出し線を使った場合、自動判定では「地番不明」になります（後述）

## 出力結果の確認ポイント

### 結線指示票

- **北西の角から時計回り**で杭番号が並びます
- 「**地番不明**」シートは、引き出し線で地番テキストが区画外にある等、自動判定できなかった区画です。図面を確認して地番を手書きで補記してください
- **公共用地**（道路・河川等）もラベルが検出された場合はシートが生成されます
- 100点を超える区画は自動的に2ページに分割されます

### 交点計算指示書

- **同一直線ペア（基準線）が1つ + 方向点が1つ**の明確なケースのみ生成されます
- 複数の直線が交わる複雑な交点は自動生成されません（人手で対応）
- 基準線の左右は方向点から見た位置関係で決定されています

## 処理の仕組み

```
DXFファイル
  │
  ├─ dxf_parser.py ─── 杭(CIRCLE)・境界線(LWPOLYLINE)・地番(MTEXT)を抽出
  │                     杭番号↔CIRCLE のマッチング（ハンガリアン法）
  │                     境界線グラフ（networkx）を構築
  │
  ├─ kessen_generator.py ─── 平面グラフの面（フェイス）探索
  │                          各地番テキスト → Point-in-Polygon でマッチング
  │                          北西角から時計回りに杭番号を並べる
  │
  ├─ kouten_generator.py ─── 交点杭の隣接ノードから同一直線ペアを検出
  │                          基準線・方向点を判定
  │
  └─ excel_writer.py ─── テンプレートに書き込み、Excel出力
```

## テンプレートのカスタマイズ

`data/templates/` にブランクテンプレートが入っています。

- `結線指示票 - ブランク.xls` — 杭番号は行5~29、列B/E/H/K に書き込まれます
- `交点計算指示書 - ブランク.xlsx` — 3列×5段のブロック配置

自治体や事業体の様式に合わせて、テンプレートを差し替えることができます。セルの位置を変更する場合は `excel_writer.py` のセル位置定数を調整してください。
